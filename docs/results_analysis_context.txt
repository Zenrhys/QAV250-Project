SLIDE 3: "Controller Design & Expected Performance"
----------------------------------------------------------------------
PURPOSE: Show the controller architecture and what performance we expect.

FIGURE 1 — Screenshot: Controller subsystem from Quadcopter_controller model
  Cascaded PID architecture: Position → Attitude → Rate → Motor mixer

FIGURE 2 — Projected plot: Step response comparison (custom PID vs PX4 default)
  * 3 subplots: Roll, Pitch, Altitude
  * Reference (dashed), Custom (blue), PX4 Default (red)
  * Caption: "Design target: ≤ 10% overshoot, faster settling than PX4 default"

FIGURE 3 — Projected plot: Waypoint tracking (top-down 2m square)
  * Reference path (dashed square) vs actual path (solid)
  * Caption: "Expected tracking RMS < 0.2 m"

TABLE — Controller performance targets:
  | Metric            | PX4 Default | Our Target | Technique          |
  |-------------------|-------------|------------|--------------------|
  | Rise time         | ~0.3 s      | ≤ 0.3 s    | Closed-Loop PID    |
  | Overshoot         | ~15%        | < 10%      | Autotuner          |
  | Settling time     | ~1.2 s      | < 1.0 s    | Autotuner          |
  | Position tracking | ~0.3 m RMS  | < 0.2 m    | Gain scheduling    |
  | Battery robustness| Degrades    | Consistent | Gain scheduling    |

SPEAKER NOTES:
  "Our controller uses Simulink's Closed-Loop PID Autotuner to systematically
  tune all 8 control loops, followed by gain scheduling on battery voltage
  to maintain performance as voltage drops during flight. Design targets are
  based on PX4 default performance as the baseline to beat."
